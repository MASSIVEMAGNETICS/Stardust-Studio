## How It Works (The Alchemical Phases)

1. **Nebula Parsing** — The prompt is inhaled. An LLM extracts entities, relationships, constraints, and success criteria. Output: a structured "Intent Constellation" (JSON).
2. **Constellation Design** — The alchemist designs the high-level architecture as a living system: layers, data flows, failure modes, and growth vectors. It chooses patterns the way a forest chooses its canopy structure.
3. **Fragment Crystallization** — For every node in the constellation, a dedicated LLM invocation grows the actual code file. Each file is born with its own "genetic" docstring explaining why its shape is inevitable.
4. **Harmonic Integration** — All fragments are brought into orbital resonance. Imports are wired, APIs are aligned, the main entry point is composed. Dissonance (missing pieces, circular imports) is detected and resolved.
5. **Resonance Testing** — The system is placed in a vacuum chamber. Syntax is verified, basic execution smoke-tested, and an LLM auditor performs a "code review from the future" suggesting refinements. Self-healing loops run up to 3 times.
6. **Supernova Release** — The finished system is packaged. You may preview every file, then with one click push the entire living product to a new GitHub repository as a perfect, ready-to-clone star.

The entire pipeline is observable. You watch the temperature drop. You see the crystals form.

## Quickstart

```bash
git clone https://github.com/MASSIVEMAGNETICS/Stardust-Studio.git
cd Stardust-Studio
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your OPENAI_API_KEY (or any OpenAI-compatible provider) and optional GITHUB_TOKEN
streamlit run stardust_studio.py
```

Then simply:

1. Enter any natural language description of the software you want to exist.
2. Click **"Transmute Prompt → Living Product"**
3. Watch the cosmos condense.
4. When the star is born, click **"Release to GitHub"** to birth it publicly (or privately).
